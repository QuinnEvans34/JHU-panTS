# M3A2: Daily Check-Ins (Week 3)

> Due Saturday by 11:59pm · Points: 10 · Submit via GitHub · Available until Jul 25

## Overview

Consistent, honest daily reflection is a professional habit, not busywork. `standup-log.md` is a running record of how the work went, what slowed it down, and how challenges were handled. It lives in the repository, is reviewed during 1-on-1 check-ins, and is referenced at the final presentation. Treat it as a document you would be comfortable showing an employer.

## Instructions

Update `standup-log.md` at the end of every weekday (Mon–Fri). Three to five sentences per entry, honest about what actually happened (not what you wish you had done).

Standard entry:

```
## [Week X — Day] — [Date]
**Worked on:** What did you actually work on today?
**Up next:** What are you working on in the next session?
**Blockers:** Any challenges or blockers, or "None."
```

1-on-1 retrospective (Weeks 1, 3, 5 only — replaces one standard entry, written after the check-in):

```
## [Week X — 1-on-1 Retrospective] — [Date]
**What we discussed:** key topics covered in the check-in.
**Feedback received:** specific feedback or direction from the instructor.
**Action items:** what you will do differently or prioritize as a result.
**Reflection:** one or two sentences — anything that surprised you or that you want to think more about.
```

Note: Week 3 HAS a 1-on-1 retrospective. So Week 3 is four standard daily entries plus one retrospective entry (five total). The Week 3 check-in focus is experiment progress, model selection reasoning, and plan health.

## File Location & Format

`standup-log.md` in root `docs/`, chronological. Do not edit or delete previous entries. The log grows across all 5 weeks.

| Criterion | Description | Pts |
