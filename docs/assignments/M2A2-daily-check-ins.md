# M2A2: Daily Check-Ins (Week 2)

> Due Saturday by 11:59pm · Points: 10 · Submit via GitHub · Available until Jul 18

## Overview

Same as Week 1: `standup-log.md` is a running, honest record of how the work went. Reviewed during 1-on-1 check-ins and referenced at the final presentation. Treat it as something you would show an employer.

## Instructions

Update `standup-log.md` at the end of every weekday (Mon–Fri). Three to five sentences per entry, honest about what actually happened.

Standard entry:

```
## [Week X — Day] — [Date]
**Worked on:** What did you actually work on today?
**Up next:** What are you working on next session?
**Blockers:** Any challenges/blockers, or "None."
```

1-on-1 retrospective (Weeks 1, 3, 5 only — replaces one standard entry):

```
## [Week X — 1-on-1 Retrospective] — [Date]
**What we discussed:** ...
**Feedback received:** ...
**Action items:** ...
**Reflection:** ...
```

Note: Week 2 has no retrospective (those are Weeks 1, 3, 5). So Week 2 is five standard daily entries. The Week 3 check-in focus is experiment progress, model selection reasoning, and plan health.

## File location

`standup-log.md` in root `docs/`, chronological, never edit or delete past entries.

## Rubric (10 pts)

| Criterion | Description | Pts |
|-----------|-------------|-----|
| Consistency & Timeliness | All five daily entries present, dated correctly, no gaps. | 4 |
| Honesty & Specificity | Entries describe what actually happened, including slow days, blockers, pivots, specific enough to understand exactly what was done. | 3 |
| 1-on-1 Retrospective (Weeks 1, 3, 5 only) | Not applicable to Week 2. | 3 |
